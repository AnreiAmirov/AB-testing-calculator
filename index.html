Test parameters

Metric → Test reference

A quick map from what you measure to which test you should run.

Conversion / proportion → Two-proportion Z-test

What it’s for: Any binary outcome per user — did they convert, click, sign up, retain? One row per user, one outcome (1 or 0).

Formula: Z = (p₂ − p₁) / √(p̄(1−p̄)(1/n₁ + 1/n₂)) where p̄ is the pooled rate.

Requirement: at least 10 conversions per arm for the normal approximation.

Examples: checkout conversion, email open rate, click-through rate, signup rate, day-7 retention.

Continuous per-user → Welch’s t-test

What it’s for: Numeric metrics with one value per user — revenue, session time, items added. Welch’s handles unequal variances correctly (always use it instead of Student’s).

Formula: t = (x̄₂ − x̄₁) / √(s₁²/n₁ + s₂²/n₂). Welch-Satterthwaite df.

Watch out: heavy-tailed metrics (ARPU, LTV) often have whales dominating. Winsorise at p99 and use CUPED to reduce variance 30–50%.

Examples: ARPU, average session length, items in cart per user, 14-day spend.

Ratio metrics → Delta method + Z-test

What the delta method is: A variance estimation technique — not a separate statistical test. It computes the correct standard error for a ratio metric, which is then used in a standard Z-test.

The full flow for ratio metrics:

  1. Aggregate to the user level: each user contributes one (X, Y) pair
  2. Use the delta method to compute the correct per-user variance of R = mean(X)/mean(Y)
  3. Apply a standard Z-test: Z = (R_B − R_A) / √(Var(R_A) + Var(R_B))

The trap (why naive doesn’t work): Pooling all orders and running a t-test treats each order as independent. Two orders from the same user are correlated — the naive SE is 2–10× too small, inflating Type I error to 10–30%.

Delta-method variance formula:
Var(R) ≈ (1/n)·[Var(X)/Ȳ² − 2·X̄·Cov(X,Y)/Ȳ³ + X̄²·Var(Y)/Ȳ⁴]
where n = number of users (not orders), X = per-user numerator, Y = per-user denominator.

Examples: AOV, CTR per session, items per order, revenue per session.

Skewed / small n → Mann-Whitney U

What it’s for: Heavily skewed metrics with small samples (n < 200), where CLT is shaky and outliers dominate.

What it tests: stochastic dominance, NOT mean equality. Report medians as the natural summary.

Examples: B2B contract value (ACV), enterprise deal sizes, early-stage metrics.

3+ variants → χ² test

What it’s for: A vs B vs C on a binary or multi-category outcome. Avoids multiple-comparison inflation of pairwise tests.

Workflow: run χ² across the contingency table. If significant, pairwise Z with Bonferroni (α/k).

Examples: 3 onboarding flows, 4 subject lines, 5 pricing tiers.

How α, β, and power interact

Type I and Type II errors are tightly coupled. Power = 1 − β.

LeverαβPowerMechanism
Increase n—↓↑SE shrinks like 1/√n
Reduce variance (CUPED, stratification)—↓↑Same as more data, achieved statistically
Raise α (0.05 → 0.10)↑↓↑Lower rejection threshold
Lower α (0.05 → 0.01)↓↑↓Higher rejection threshold
One-sided test—↓↑All α budget on one tail
